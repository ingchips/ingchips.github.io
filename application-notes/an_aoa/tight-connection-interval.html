Application Note: Direction Finding Solution

Chapter 4 Tight Connection Interval

4.1 Overview

The Link Layer in the Connection state transmits Data Physical Channel PDUs in connection events. The start of connection events are spaced regularly with an interval of connInterval and do not overlap.

In Bluetooth Core Specification, the connInterval is defined to be a multiple of \(1.25 ms\) in the range \(7.5 ms\) to \(4.0 s\).

Tight Connection Interval is introduced to extend the range to \(1.25ms\) to \(4.0s\). With the new minimum \(1.25ms\) interval, extreme low latency can be achieved.

Interval can further be reduced by customizing interval unit through Link Layer API ll_set_conn_interval_unit.

4.2 Usage

  1. Create the peripheral & central app against the extension bundle;

  2. When calling gap_ext_create_connection to create the connection, set both interval_min and interval_max to \(1\) in PHY configurations.

4.3 Compatibility

Both peripheral & central must support this extension.
