使用 SWD 接口进行产品量产

ING918 系列芯片支持通过 SWD 接口进行批量烧写。工具可以直接使用 IDE 或者专门的 JTAG/SWD 工具。本文 以 Keil 和 J-Link 为例介绍具体操作步骤。

使用 Keil 量产

说明: 以下步骤介绍了烧写用 Keil 项目的具体创建方法，Keil μVision 4 或 5 皆适用。 用户可以按照步骤自行创建或者直接 下载(镜像)我们提供的示例。

用来烧写 platform 的空白项目

  1. 创建空白项目

    在 Keil 里新建一个项目，CPU 选为 INGCHIPS -> ING91800。项目创建完成后，确认不包含任何源代码（C 或汇编），如果有， 则将其删除。

    修改项目的属性：

    • 将输出的可执行文件改为 platform.hex （注意扩展名为 .hex）

    • 检查 Flash 烧写算法设置是否正确

      ING918 烧写算法已将 Erase 操作内置于 Program，所以不需要勾选“Erase”；一般情况下，为了提高量产速度，“Verify” 也可以不勾选。其它各设置的建议值如下图。

    • 确认选择了 SWD 端口而非 JTAG 端口

      ING918 只有 SWD 端口，无 JTAG 端口。

  2. 准备 platform.hex

    使用 hex 转换工具（如 IntelHex）将要写入产品的 platform.bin 转换为 platform.hex。

    假设创建的空白项目全名为 “C:\project\platform\platform.uvproj”，将 platform.hex 复制到 “C:\project\platform” 目录下。

  3. 烧写

    在 Keil 里直接点击 Flash Download 按钮即可完成烧写。如果出现错误，请对照检查项目设置是否有误。

用来烧写启动地址的空白项目

按相同的步骤重新创建一个空白项目。该项目用来烧写程序的启动地址：在 0x00087FFC 处写入 32bit 数值 0x00004000。 可参考 示例 里的 startaddr 项目。

量产

量产时通过命令行调用 Keil 实现批量烧写:

C:\Keil\UV4\UV4 -f C:\project\platform\platform.uvproj
C:\Keil\UV4\UV4 -f C:\project\startaddr\startaddr.uvproj

使用 J-Link 量产

手动添加芯片型号

说明: SDK 安装包暂时不能自动为 J-Link 添加 ING918 芯片支持包，用户需要手动完成。

假设 SDK 安装目录为 /sdk/flashSEGFlash，J-Link 安装目录为 /SEGGER/JLink。

  1. 更新 JLinkDevices.xml

    将 /sdk/flashSEGFlash/JLinkDevices.xml 文件里 <Device> 和 </Device> 之间的一段内容粘贴到 /SEGGER/JLink/JLinkDevices.xml 文件末尾 </Database> 之前。

  2. 复制 Flash 烧写算法包

    将 /sdk/flashSEGFlash/ 目录下 Ingchips 文件夹复制到 /SEGGER/JLink/Devices 目录。

创建 J-Flash 项目

打开 J-Flash，新建一个项目，Target Device 选为 ING9188xx。

将要写入产品的 platform.bin 拖入 J-Flash 界面，地址设为 0x0004000。点击 Target -> Connect 菜单， 然后点击 Target ->Manual Programming -> Program，即可将 platform.bin 烧写到产品。同法可以烧写 启动地址。

说明: J-Flash 需要注册方可使用全部功能。

合并数据

为了量产方便，通过 File -> Merge data file 菜单将前文挺到的将 platform.bin、startaddr.bin 以及产品固件合并 为一个 hex 数据文件。
