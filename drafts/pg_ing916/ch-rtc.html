17 实时时钟（RTC）

17.1 功能描述

实时时钟是一个独立的定时器。RTC模块拥有一组连续计数的计数器，在相应的软件配置下， 可提供时钟日历的功能。修改计数器的值可以重新设置系统当前的时间和日期。

特性：

  • 用于寄存器访问的AMBA 2.0 APB接口
  • 可配置计数器大小
  • 周期性中断包括半秒、秒、分钟、小时和天
  • 可编程报警中断
  • 硬件数字微调可补偿外部时钟源的不准确性

17.2 使用说明

17.2.1 RTC使能

使用 RTC_Enable 使能RTC。

void RTC_Enable(
  uint8_t enable  // 使能(1)/禁用(0)
  );

17.2.2 获取当前时间

使用 RTC_GetTime 获取当前时间，该函数的参数可以读取对应的时、分、秒， 函数的返回值即为RTC使能后所经历的天数 day。

uint16_t RTC_GetTime(
  uint8_t *hour,    //时
  uint8_t *minute,  //分
  uint8_t *second   //秒
  );

17.2.3 修改时间

使用 RTC_ModifyTime 修改当前时间，其中的 day 表示RTC使能后所经过的天数。

void RTC_ModifyTime(
  uint16_t day,   //天数
  uint8_t hour,   
  uint8_t minute, 
  uint8_t second
  );

修改时间后，需要使用 RTC_IsModificationDone 获取当前修正是否已经同步到RTC时钟域， 返回值为 1 表示已经同步完毕，0 则表示正在更新。

int RTC_IsModificationDone(void);

17.2.4 获取RTC Counter值

使用 RTC_Current 可以获取RTC Counter值的低32bit。

uint32_t RTC_Current(void);

使用 RTC_CurrentFull 可以获取RTC Counter值的全部43bit。

uint64_t RTC_CurrentFull(void);

17.2.5 配置闹钟

使用 RTC_ConfigAlarm

void RTC_ConfigAlarm(
  uint8_t hour, 
  uint8_t minute, 
  uint8_t second
  );

17.2.6 配置中断请求

使用 RTC_EnableIRQ 配置并使能RTC中断请求。

void RTC_EnableIRQ(
uint32_t mask   //比特 2 为 1 时使能alarm中断
                //比特 3 为 1 时使能day中断
                //比特 4 为 1 时使能hour中断
                //比特 5 为 1 时使能minute中断
                //比特 6 为 1 时使能second中断
                //比特 7 为 1 时使能half-second中断
);

RTC支持的中断类型一共6种：

typedef enum
{
    RTC_IRQ_ALARM = 0x04,       //alarm中断
    RTC_IRQ_DAY = 0x08,         //day中断
    RTC_IRQ_HOUR = 0x10,        //hour中断
    RTC_IRQ_MINUTE = 0x20,      //minute中断
    RTC_IRQ_SECOND = 0x40,      //second中断
    RTC_IRQ_HALF_SECOND = 0x80, //half-second中断
} rtc_irq_t;

上述所提到的六种中断中，alarm中断通过调用 RTC_ConfigAlarm 设定alarm时间， 其余中断的出发时间均是固定的，依次为：1天、1小时、1分钟、1秒钟和半秒钟。

  • 例如使能 RTC 设的alarm中断，并设定alarm时间的01:45:20

    RTC_EnableIRQ(RTC_IRQ_ALARM);
RTC_ConfigAlarm(0x1,0x2d,0x14);
RTC_Enable(1);

  • 例如使能 RTC 的day中断

    RTC_EnableIRQ(RTC_IRQ_DAY);
RTC_Enable(1);

17.2.7 获取当前中断状态

使用 RTC_GetIntState 获取当前RTC的中断状态。

uint32_t RTC_GetIntState(void);

17.2.8 清除中断

使用 RTC_ClearIntState 清除当前RTC的中断状态。

void RTC_ClearIntState(
  uint32_t state    //比特 2 为 1 时清除alarm中断
                    //比特 3 为 1 时清除day中断
                    //比特 4 为 1 时清除hour中断
                    //比特 5 为 1 时清除minute中断
                    //比特 6 为 1 时清除second中断
                    //比特 7 为 1 时清除half-second中断
  );

17.2.9 处理中断状态

用 RTC_GetIntState 获取RTC上的中断触发，返回非 0 值表示 RTC 上产生了中断请求。 RTC产生中断后， 需要消除中断状态方可再次触发。利用 RTC_ClearIntState可清除RTC的中断触发状态。

17.2.10 睡眠唤醒源

RTC 支持作为低功耗状态的唤醒源：RTC的闹钟唤醒信号在 RTC_ConfigAlarm 指定的时间将系统唤醒。 使用 RTC_EnableDeepSleepWakeupSource 使能 RTC 的唤醒功能。

void RTC_EnableDeepSleepWakeupSource(
uint8_t enable    //使能(0)/禁用(1)
)

17.2.11 数字调校

__硬件数字微调ATCRTC100主要由外部32768Hz时钟源驱动，其脉冲发生器产生内部1Hz脉冲以增加其时间计数器。 1Hz 脉冲是通过计算外部时钟源的时钟周期数产生的。缺省情况下，脉冲发生器每32768个时钟周期输出一个周期宽的脉冲。 因此，如果外部时钟源恰好是32768Hz, 1Hz脉冲的周期将是一秒，第二计数器每秒钟增加一次。但是，如果外部时钟周期不准确， 所得到的周期就不会恰好是一秒。所提供的数字微调功能通过增加或减少产生1 Hz脉冲的周期来补偿这些不准确性。

  1. 设置单个修正值

  • 设置方向，通过 rtc_trim_direction_t 中的枚举变量设置修正的方向，即修正值的正负，从而使得时钟加快或者减慢。

    typedef enum
{
    RTC_TRIM_FAST = 0x00,  
    RTC_TRIM_SLOW = 0x01,
}rtc_trim_direction_t;

  • 设置 秒修正值 ，作用于每秒对应的计数值。

    void RTC_SecCntTrim(
  uint8_t trim,      //修正值（计数值）
  rtc_trim_direction_t dir  //方向，具体参考 `rtc_trim_direction_t`
  );

  • 设置 分钟修正值 ，作用于每分钟最后1秒的计数值。

    void RTC_MinCntTrim(
  uint8_t trim, 
  rtc_trim_direction_t dir
  );

  • 设置 小时修正值 ，作用于每小时最后1秒的计数值。

    void RTC_HourCntTrim(
  uint8_t trim, 
  rtc_trim_direction_t dir
  );

  • 设置 天修正值 ，作用于每天最后1秒的计数值。

    void RTC_DayCntTrim(
  uint8_t trim, 
  rtc_trim_direction_t dir
  );

注意，若设置多个修正值，则以较长时间段的修正值为优先。譬如，设置了 秒修正值 和 分钟修正值 ， 则 秒修正值 只作用于每分钟的前 59 秒，最后 1 秒则采用 分钟修正值 ， 其余情形以此类推。

  1. 设置多个修正值

通过 RTC_SetAllTrimValue 可以同时设置多个时间段的修正值。

void RTC_SetAllTrimValue(
  uint8_t day_trim, 
  uint8_t hour_trim, 
  uint8_t min_trim,
  uint8_t sec_trim
  );
需要注意的是，8bits中只有低5bits用于表示修正值，最高bit表示修正的方向，也就是正负，如下图所示（图 17.1）。：

图 17.1: RTC设置多个修正值数据结构

  1. 获取所有修正值

通过 RTC_GetAllTrimValue 可以获取多个时间段的修正值，返回值为 day_trim ，获取到的值的数据结构与 RTC_SetAllTrimValue 中所使用的结构一致， 具体可参照上图17.1。

uint8_t RTC_GetAllTrimValue(
  uint8_t *hour_trim, 
  uint8_t *min_trim,
  uint8_t *sec_trim
  );
  1. 清除所有修正值

通过 RTC_ClearAllTrimValue 可以将所有修正值置0;

void RTC_ClearAllTrimValue(void);
